# 超值限时组合包购买决策规划需求

本文档记录购买规划器需要满足的产品行为和用户可见规则。游戏内稳定规则见 `game-rules.md`；算法、性能策略和实现取舍写入 `implementation-design.md`。

本文只描述规划器如何基于游戏规则、玩家预期性价比、机会偏好和计算前瞻范围，判断当前哪些礼包值得买、哪些机会值得触发、哪些机会应保留。

---

## 1. 需求定位

购买规划器不是预算花费器，也不是触发机会消耗器。

玩家不是给规划器一笔钱让它花完，而是给规划器一个“什么样的购买值得”的标准；玩家也不是给规划器一堆机会让它用完，而是给规划器一个当前起点，让规划器判断哪些机会现在值得用、哪些机会应留到未来。

核心目标：

```text
基于预期性价比、机会偏好和当前触发起点，
在计算前瞻范围内生成购买、触发、跳过和保留决策。
```

规划器允许输出“暂不购买 / 暂不触发 / 保留机会”。如果当前没有达到玩家标准的路径，空方案是有效方案。

---

## 2. 核心概念

### 2.1 购买意愿与机会偏好 (Purchase Strategy & Preference)

购买意愿表示玩家对“花钱换收益的门槛”以及“触发机会是否值得现在用”的综合倾向。

在纯粹的 CE 模型中，预期性价比与机会成本在数学底层是高度重叠的约束。规划器摒弃了独立的数字输入，转而提供直观的宏观档位选择。这二者在底层决策模型中呈强相关的映射：

| 偏好档位 | 含义 | 行为倾向 |
| --- | --- | --- |
| 激进 (预算充足) | 愿意接受较低的性价比以换取绝对收益 | 算法倾向于通过升档、跨批次合并获取高价包。由于机会成本不敏感，很容易触发并购买。 |
| 均衡 (默认) | 兼顾性价比与进度推进 | 以硬通货 5900 付费钻档的动态 CE 基准寻找全局最优。 |
| 保守 (极其挑剔) | 预算有限，只买最划算的特惠包 | 极高的盈亏线，宁可暂停进度保留机会，也绝不为了推进而去买普通收益包。 |

偏好档位的 CE 门槛不使用固定常数，而是从当前评分表计算出的礼包 CE 中推导：

- 基准优先取 5900 付费钻限时包中，包含标准硬通货的礼包 CE 中位数。标准硬通货为：命运召唤券、圣天使的神谕召唤券、星愿礼券袋、星虹礼券袋。
- 如果当前数据中没有标准硬通货 5900 档样本，再用次级硬通货样本兜底。次级硬通货为：魔女的奥秘召唤券、生命树之露、禁忌武具召唤券。
- 若两类样本都不存在，才回退到旧的 `CE = 3.0` 默认基准。
- 内部映射为：保守 `基准 × 1.25`，均衡 `基准 × 1.0`，激进 `基准 × 0.75`，自定义则直接使用玩家填写的 CE。若未单独设置 `strategyCeThreshold`，策略筛选门槛沿用该 `expectedRatio`。
- 页面需要同时展示基准 CE 和当前门槛 CE。基准 CE 是由硬通货礼包数据推导出的市场锚点；当前门槛 CE 是本次偏好真正参与路径盈亏判断、补累充判断和策略筛选的买入线。

由于引擎严格遵循该策略计算出一条最大净收益的最优路径，规划器不再设立“安全花费上限（硬性主预算）”。玩家只需沿着最优路径走到自己认为合适的开销点停止即可。

### 2.3 计算前瞻范围

计算前瞻范围表示规划器本次向后纳入比较的触发机会范围，用于判断当前机会的机会成本、保留价值和后续路径收益。

计算前瞻范围参与计算，但不是玩家必须推进到的位置，也不是最终必须展示的范围。

页面应区分：

| 字段 | 含义 | 是否参与计算 |
| --- | --- | --- |
| 计算前瞻范围 | 纳入路径搜索与机会成本比较的未来范围 | 是 |
| 展示范围 / 展示明细深度 | 页面展示多少批次、节点和明细 | 否 |

推荐交互为“内置默认前瞻范围 + 玩家可编辑覆盖”。默认范围应覆盖常见的一段推进目标，降低新手配置成本；玩家填写范围用于表达真实计划边界，例如本次只推到某一主线章节、塔层或等级段。只使用内置固定范围会让规划器把玩家不打算触发的未来机会纳入机会成本；只依赖玩家手填则容易配置负担过重。两者结合更符合当前功能。

前瞻范围不是显示裁剪。扩大前瞻范围会改变搜索空间、机会成本、升降档判断、跨日补累充判断和中间批次选择，因此必须重新计算。

## 3. 输入

页面字段：

- CE 门槛：保守 / 均衡 / 激进 / 自定义。决定算法底层的盈亏线及机会摩擦成本，页面需要标注每档对应的实际 CE；自定义档允许玩家直接填写 CE。
- 当前礼包档位：玩家当前已处于的限时组合包档位。
- 各触发源当前进度：玩家当前已达到的位置；规划器只考虑大于该值的触发点。
- 起始触发机会：从当前进度出发，下一批可能被安排触发的机会集合；可由当前进度和礼包触发表自动推导，也可在高级模式下手动指定。
- 计算前瞻范围：规划器向后纳入比较的触发机会范围。
- 展示范围 / 展示明细深度：只影响页面展示，不影响计算。
- 2 小时内最多触发：同一触发源在同一个卡包批次内最多能连续触发多少个限时包；这是同批上限，不要求必须合并到上限。
- 补累充模式：关闭 / 自动价值判断 / 手动确认。

输入格式：

- 主线进度支持 `章节-关卡`，例如 `13-28`；也支持游戏内连续关卡进度，例如 `336`。
- 其他来源使用数字层数或等级。
- “全属性塔抵达”由四个属性塔自动派生，页面不单独提供起始层、目标层或每批触发数。

字段替换关系：

| 旧字段 | 新字段 | 调整原因 |
| --- | --- | --- |
| 预期性价比 / 安全花费上限 | CE 门槛 | 将底层的绝对标尺合并为基于硬通货基准 CE 的自动门槛，不再设立硬性封顶花费 |
| 补累充阈值 | 补累充模式 + 价值判断 | 补累充也是购买行为，应进入性价比评价 |
| 计划推到 | 计算前瞻范围 | 表示计算视野，不是推进目标 |
| 当前触发机会数量 | 起始触发机会 | 表示当前规划起点，不表示必须消耗 |
| 价值最优 | 决策价值最高 | 同时考虑钱、机会、执行成本和未来状态 |

## 4. 路径价值模型

### 4.1 路径花费与价值

为保持核心规划的纯净度，规划器主路径的花费与价值**仅包含限时组合包**：

```text
路径总花费 = 限时组合包花费
```

```text
路径总价值 = 限时组合包内容 CE + 实际每日累充新增赠钻 CE
```

*注意：如果系统建议玩家补包，补包的花费和额外价值会作为独立的明细和状态分支纳入规划，并合并计入 DP 的 `totalSpentYen` 账本；是否补包由 `topUpSurplus >= 0` 和状态价值比较决定，不再依赖安全预算阈值。*

### 4.2 金钱净收益

```text
moneySurplus =
路径总价值 - (路径总花费 / 2) × expectedRatio
```

`moneySurplus >= 0` 表示路径在金钱效率上达到玩家预期。

金钱净收益仍是核心指标，但不能作为唯一排序标准，因为它没有完整体现触发机会的保留价值。

### 4.3 机会状态价值

规划器应评价“动作后的状态”，而不是只评价“已买路径”。

```text
stateValue =
从某个状态继续规划时，在计算前瞻范围内可获得的最佳决策价值
```

状态至少包括：

- 当前礼包档位。
- 各触发源前沿。
- 已触发 / 已购买 / 已保留节点。
- 当前累充日累计付费钻。
- 当前累充日已推进过的属性塔集合。
- 剩余可达触发机会。
- 是否可跨日重置。
- 机会偏好。

机会成本通过反事实比较体现：

```text
机会成本 =
保留该机会后的最佳状态价值
- 消耗该机会后的最佳状态价值
```

如果保留机会未来更优，则当前消耗机会的成本高；如果机会没有更好用法，则成本低。

### 4.4 决策价值

规划器在搜索扩展时以决策价值（decisionValue）作为剪枝优先级指标，但在最终方案输出时以真实收益（realScore）为主排序指标，以防启发式过高估导致规划器提前停滞。

```text
decisionValue =
moneySurplus
+ remainingStateValue
- executionCost
- riskCost
```

也可以用搜索 / 动态规划形式表达：

```text
推荐动作 =
argmax(当前动作收益 + 后续状态最佳价值)
```

其中：

- `moneySurplus` 判断钱是否花得值。
- `remainingStateValue` 判断机会是否值得现在用。
- `executionCost` 表示同批触发压力、批次数、触发但不买、跨日复杂度等执行成本。
- `riskCost` 表示补累充、较高总花费、复杂操作等风险成本。

排序优先级：

1. 决策价值最高。
2. 金钱净收益更高。
3. 关键机会消耗更少。
4. 实际性价比更高。
5. 执行压力更低。
6. 补累充风险更低。
7. 总花费更低。

## 5. 触发机会与动作

规划器处理的触发源包括：

- 主线
- 玩家等级
- 无穷塔
- 蓝塔 / 红塔 / 翠塔 / 黄塔
- 全属性塔抵达

每个触发机会可处于以下状态：

- 可达：在当前前沿、计算前瞻范围和拓扑约束下，理论上可安排触发。
- 本批触发：被选入当前候选批次，视为已触发，并推进对应来源前沿。
- 已购买：本批触发节点中实际购买的限时包。
- 触发未买：本批触发但不购买，批内全不买时下一批降档。
- 保留：当前可达但未触发，不推进来源前沿，可留到后续批次或未来规划。

可选动作：

- 保留机会。
- 触发并购买。
- 触发但不购买，用于降档或跳过低价值档位。
- 同批部分购买。
- 合批或拆批。
- 跨日重置累充。
- 自动补累充或手动确认补累充。

默认原则：

```text
如果当前触发机会不能带来高于保留机会的决策价值，
应优先保留，而不是为了消耗机会而触发。
```

## 6. 拓扑约束

规划器采用玩家主动控制触发时机的“乐观卡包”假设：玩家可以控制主线、无穷塔、属性塔等来源的推进节奏，在限定范围内把触发机会安排到一个或多个 2 小时卡包批次中。

乐观卡包不代表可以任意打乱触发顺序。

### 6.1 普通触发源

同一触发源内必须保持顺序，不能把后续触发点排到前置触发点之前。

### 6.2 属性塔

蓝塔、红塔、翠塔、黄塔各自维护独立前沿。某个单塔节点被选入本批触发节点后，只推进该塔自己的前沿；未选入当前批次的同层单塔节点仍可保留。

同层不同单塔之间没有直接先后关系。可达节点可以合批，也可以拆批；批次编排只需满足：

- 同一来源自身顺序。
- 同批触发上限。
- 全属性塔抵达派生门槛。

### 6.3 全属性塔抵达

全属性塔抵达由四塔进度共同派生：

```text
全属性塔进度 = min(蓝塔层数, 红塔层数, 翠塔层数, 黄塔层数)
```

只有四塔都达到某个全属性塔抵达礼包层级时，该节点才可触发。

在规划器前沿模型中，判定方式为检查每个单塔前沿区间是否覆盖该 allTower 层：

```text
previous < allTowerLayer <= next
```

任一塔的前沿区间未覆盖该层时，该全属性塔抵达节点不可达。

全属性塔抵达节点本身不推进蓝塔、红塔、翠塔或黄塔的单塔前沿；它只是由四塔共同派生出的触发机会。

触发全属性塔抵达时，四塔中“最后达到该层”的塔不能在同一批次中同时触发其下一单塔节点。该塔刚跨过全属性塔门槛，尚未越过该门槛后的单塔触发栅栏。

## 7. 批次与升降档

一个全局结算批次表示一个 2 小时卡包窗口。不同触发源若能通过控进度安排到同一 2 小时窗口，可以进入同一个全局结算批次。

合批与拆批规则：

- 可同批不代表必须同批。
- 可拆批不代表必须拆批。
- 同一来源在同批内触发次数不能超过“2 小时内最多触发”。
- 合批不按固定来源批次序号决定，只要满足来源顺序、同批上限和属性塔 / 全属性塔约束即可。
- 同批触发数越多，执行压力越大；在决策价值相近时，优先选择触发数更少、批次压力更低的方案。

升降档规则：

- 批内至少购买 1 个限时包，则下一批升档。
- 批内全不买，则下一批降档。
- 批内部分购买时，只要至少购买 1 个，批内未购买礼包不会额外导致降档。
- 档位不低于最低档，不高于最高档。

批次编排会影响：

- 升降档次数。
- 后续可购买档位。
- 同一累充日内累计付费钻。
- 是否跨 0 点重置累充。
- 是否可补累充。
- 触发机会被消耗还是保留。

因此批次编排属于购买决策的一部分，不是单纯展示分组。

## 8. 累充日模型

规划器不要求用户填写真实日期、开始时间或规划天数，使用抽象累充日处理：

- 批次顺序。
- 2 小时超时。
- 0 点累充重置。
- 属性塔推进带来的强制累充日约束。

不同批次可以共用同一累充日进度，也可以在批次之间跨 0 点并重置每日累充进度。每日累充重置只在两个全局批次之间发生；同一全局批次内的限时包购买和补累充购买视为发生在同一累充日内。

### 8.1 属性塔强制重置

记：

```text
A = 当前累充日内已经推进过的单属性塔集合
B = 当前候选批次中被选为本批触发节点的单属性塔集合
```

判定规则：

- 若 `A ∩ B` 非空，则当前候选批次执行前必须先跨日重置累充。
- 若 `A ∩ B` 为空，则当前候选批次可以继续沿用当前累充日，并在批次执行后将 `B` 合并进 `A`。
- 首个批次 `A` 为空，不会因为属性塔推进而强制预先重置。
- 可达但未进入当前候选批次的属性塔节点，不参与 `B`。
- 不同属性塔在同一累充日内推进不强制重置。
- 同一属性塔在同一累充日内再次推进才强制重置。
- 全属性塔抵达节点本身不推进四个单塔前沿，因此不计入 `B`。

当前乐观卡包策略不使用跨 4 点额外推塔能力。若未来新增精确跨 4 点属性塔模式，则同一个 2 小时批次不能同时作为跨 0 点累充批次和跨 4 点属性塔批次。

### 8.2 每日累充赠钻

购买规划页按路径内同一累充日累计付费钻计算实际赠钻：

```text
本次赠钻 =
TotalBonusFreeDiamonds(购买后累计付费钻)
- TotalBonusFreeDiamonds(购买前累计付费钻)
```

不同批次共用同一累充日时，累计付费钻持续累加；跨日重置后，新的累充日从 0 开始计算。

每日累充赠钻 CE 必须计入路径总价值。

## 9. 自动补累充的判定分支 (Top-up DP Branching)

补累充不再作为脱离 DP 树的强制捆绑动作，而是作为跨日前的一层独立价值判定。这保证了规划器不强迫玩家购买常驻包；只有当补包在当前 CE 标准下有利可图时，才会进入推荐路径或展示为建议动作。

### 9.1 价值判定与触发机制

当规划器处理完一个批次并准备跨日（或同日无后续动作）时，会对当前累计付费钻进行扫描：

```text
topUpSurplus = 补包新增累充赠钻CE + 补包内容CE - (补包花费 / 2) * expectedRatio
```

*   如果 `topUpSurplus >= 0`：规划器可以选择“买常驻包后跨日”的状态，该状态将继承补包带来的高额性价比收益。
*   如果补包低于当前 CE 标准，规划器应保留“不补包直接跨日”的状态，不应为了凑累充而强制购买。
*   当补包与不补包跨日后的后续规划状态完全一致时，引擎可以基于决策价值执行支配剪枝，只保留更优状态。

### 9.2 机制的优势与规则

- **保持路径一致性**：由于判定发生在规划状态流转内，补包带来的额外收益会计入路径价值与展示明细，不会在 UI 后处理中被强制搭售。
- **规划模式**：默认使用长期规划模式，以每日 12000 付费钻作为规划上界。若下一次购买会越过该上界，应先在批次边界结算并跨日，而不是继续冲击 18000/24000/36000 档。玩家开启赶进度模式后，算法允许同日继续冲击 18000/24000/36000 档，并用累充包络线和跨日重置惩罚表达“时间有成本”的偏好。
- **补包组合规则**：补包只作为参考建议，不要求严格求解“最少钻 / 最低花费”的最优组合。引擎应优先性能和稳定性：先按常驻钻石组合包等次各参考一次，若仍未达到目标档，再用小档位自由补齐；允许少量超补，但不得为了小缺口推荐明显过大的低性价比补包，也不得推荐低于当前 CE 标准的补包。

## 10. 候选路径生成与停止

规划器从起始触发机会开始生成候选路径。每一步可以选择：

- 保留当前可达机会。
- 将一个或多个可达机会安排进当前批次。
- 当前批次购买全部、购买部分或全部不买。
- 当前批次后继续沿用同一累充日。
- 当前批次后跨 0 点重置累充日。
- 跨日前按补累充模式处理补累充。
- 等待 2 小时使批次超时并触发降档。

路径必须满足：

- 来源内顺序。
- 同批触发上限。
- 属性塔 / 全属性塔抵达约束。
- 升降档规则。
- 累充日规则。
- 计算前瞻范围。
- 可选安全上限。
- 补累充安全设置。

停止扩展条件：

- 后续所有动作的决策价值均无法改善。
- 继续扩展只会降低综合性价比，且无法通过升档、跨日、补累充或保留机会恢复。
- 已达到计算前瞻范围。
- 已达到可选安全上限。
- 后续没有可达触发机会。

停止表示继续买或继续触发不符合当前偏好，不是规划失败。

## 11. 方案类型

页面同时生成多种方案，供玩家按实际买包策略切换。`preferenceLevel` 只表示价值门槛与机会折现，不再承担“玩家买法”的全部语义。

### 11.1 价值最优方案

兜底推荐方案。目标是在玩家预期性价比和机会偏好下，通过决策价值指引探索，并最终选择**真实收益 (realScore)** 最高的方案。

可包含升档铺路、触发不买降档、拆批、跨日和补累充，但必须解释这些动作为什么提高决策价值。

如果全局最高 `realScore` 路径与已展示的策略方案只存在微小的触发位置、购买数或均 CE 差异，并且本质仍是同一类买法，页面可以不单独展示“价值最优”。此时对应策略方案已经代表了全局最优附近的可执行建议。

### 11.2 保档大包方案

面向“低 CE 包卡掉，但尽量保住档位，其它机会买较高档包”的玩家。

规则：

- 允许同一批触发多个礼包时只购买其中 CE 达标的礼包。
- 若同批至少购买 1 个礼包，本批未购买的低 CE 包不会导致额外降档。
- 避免整批触发后全不买导致下一批降档。
- 已购买礼包必须达到当前策略 CE 门槛。
- 路径内至少包含 1 个 1500 付费钻及以上的购买点，避免纯小包路径被误标为“保档大包”。

### 11.3 3000 付费钻微操方案

这是保档大包方案的衍生策略，不作为独立的宏观偏好。它只在 3000 付费钻档（价格 6000 档）能形成可比较收益时展示。

适用场景：

- 个别 3000 付费钻档礼包 CE 高于同触发点或相邻路径中的 5900 付费钻档。
- 玩家愿意通过主动降档或等待，把后续购买落在 3000 付费钻档。
- 该路径可减少少量补累充开销，或提升路径平均 CE。

若该方案与保档大包方案收益几乎一致，或无法形成至少两个 3000 付费钻档购买点，页面可以不展示它，以保持系统简洁。

### 11.4 小包批量方案

面向“尽量只买最小包，但会一批买多个”的玩家。

规则：

- 只购买 80 付费钻档（价格 160 档）礼包。
- 优先选择同一批能购买多个高 CE 小包的路径。
- 当策略 CE 门槛高于多数大包但仍低于 80 付费钻档高 CE 包时，该方案应自然胜出或作为可选策略出现。
- 纯 80 付费钻路径不得归入“保档大包”；如果没有形成批量小包特征，则应由价值最优兜底展示。

### 11.5 机会保留方案

当当前起点之后没有达到预期的路径，或当前使用机会不如保留机会时，生成机会保留方案。

展示：

- 当前不建议购买。
- 当前不建议继续触发的节点。
- 保留原因。
- 若降低预期性价比到多少，或调整机会偏好到什么程度，才会出现可推荐路径。
- 哪些未来状态可能让这些机会重新值得使用。

因此，方案类型并非固定全部出现：只有找到满足该买法规则、真实收益为正、且不与已展示方案策略等价的代表路径时，才输出对应卡片。若玩家调高 CE 门槛，输出可能从保档大包转向小包批量或机会保留；若调低 CE 门槛，价值最优可能覆盖更多大包和补累充路径。

## 12. 展示

### 12.1 汇总

规划结果汇总应显示：

- 推荐结论：建议购买 / 建议暂不购买 / 建议先降档 / 建议保留机会。
- 预期性价比。
- 机会偏好。
- 路径实际性价比。
- 金钱净收益。
- 决策价值。
- 总价值。
- 总花费。
- 限时包花费。
- 补累充花费。
- 购买限时包数。
- 已触发机会数。
- 已购买机会数。
- 保留机会数。
- 跨日重置次数。
- 补累充批次数。
- 下一批结束后的礼包档位。

若存在可选安全上限，应显示当前方案是否触达上限，以及因安全上限被过滤的高价值方案数量。

### 12.2 批次列表

列表层展示：

- 批次编号。
- 本批动作：购买 / 不买 / 部分购买 / 保留。
- 本批触发节点数。
- 本批购买节点数。
- 本批限时包花费。
- 本批补累充花费。
- 本批新增价值。
- 本批金钱净收益。
- 本批后累计实际性价比。
- 是否跨日重置。
- 是否补累充。补包列必须区分“已补”“跨日前已判断不补”和“同一累充日尚未跨日结算”。尚未跨日结算的批次不得简单显示“无”，避免玩家误解为该累充日无需补包。
- 本批后下一档位。
- 本批决策理由。

连续不购买的批次可以压缩展示：

```text
第 2-4 批，连续触发不买 ×3，用于降档
```

但压缩展示必须保留展开明细，避免玩家误解为机会没有被消耗。

### 12.3 展开明细

展开明细显示：

- 每个礼包的触发源与触发点。
- 触发前档位。
- 购买价格与付费钻。
- 礼包内容 CE。
- 每日累充新增赠钻 CE。
- 补累充新增价值。
- 单包性价比。
- 路径级增量性价比。
- 是否购买。
- 未购买原因。
- 是否属于保档大包、3000 付费钻微操或小包批量策略。
- 若触发后未购买，是否为同批卡低 CE 包，或整批不买降档。
- 是否建议保留。
- 补累充使用的常驻钻石组合包。

### 12.4 保留机会展示

当规划器输出“机会保留”空方案时，必须显式展示：

- 触发源。
- 触发点。
- 当前不建议触发的原因。
- 保留后可能等待的条件：档位变化、累充日重置、后续更高价值机会、预期性价比降低、机会偏好调整等。
- 是否属于关键机会。

当规划器输出了可购买路径，但该路径没有消耗完整个前瞻范围时，页面必须在路径结尾展示“未使用机会”摘要，而不是只在汇总区显示一个数字：

- 按触发源聚合剩余机会。
- 展示每个来源的剩余触发范围和机会数。
- 明确说明这些机会已在当前前瞻范围内，但没有被本推荐路径消耗。
- 提示如果玩家还计划继续推进，可以扩大前瞻范围，让更后面的机会参与下一段决策。

---

## 13. 计算触发

规划器不在输入变化时自动重算，需要用户点击“计算购买方案”。

如果用户只调整预期性价比、机会偏好或展示范围，页面可以复用上一次候选路径集合重新排序和过滤。

如果用户调整当前进度、起始触发机会、计算前瞻范围、同批触发上限、补累充模式或安全上限，则必须重新生成或重新校验候选路径。

## 14. 验收标准

### 14.1 不再花完预算

没有主预算花完目标。若当前路径低于预期性价比，规划器应输出暂不购买或保留机会。

### 14.2 不再用完机会

计算前瞻范围内存在大量机会时，规划器可以只建议使用其中一小部分，并明确展示保留机会和原因。

### 14.3 净收益不是唯一排序

净收益仍用于判断钱是否花得值。搜索扩展、剪枝和保留机会比较必须使用包含机会状态价值和执行成本的决策价值；最终推荐展示可在已超过保留基线的候选中按真实收益排序。

### 14.4 支持同批卡低 CE 包但必须保档语义正确

若同一批触发多个礼包且至少购买 1 个，未购买的低 CE 包不应被展示为导致降档；页面必须展示哪些包被购买、哪些包被卡掉。

### 14.5 支持 3000 付费钻与小包策略但避免无意义噪音

3000 付费钻微操方案只有在形成可比较收益时展示；小包批量方案只应购买 80 付费钻档，且至少存在一批购买多个小包。若推荐整批不买以降档，仍必须展示被消耗的触发机会、机会成本和增量收益。

### 14.6 补累充逻辑融合于主线规划

补累充不再脱离 DP 计算或盲目绑定，而是作为跨日前的状态价值判定。引擎会根据 `topUpSurplus >= 0` 判断是否值得补包；如果补包与不补包跨日后的后续状态完全一致，可以基于决策价值只保留更优状态。规划器不得推荐低于当前 CE 标准的补包，也不得为了凑累充强制搭售。

### 14.7 前瞻范围参与计算但不是目标

计算前瞻范围用于判断机会成本和未来状态价值，不表示玩家必须推进到该范围终点。

### 14.8 空方案是有效方案

如果所有候选路径都低于预期，或当前使用机会不如保留机会，规划器必须输出有效空方案：

```text
当前不建议购买；
当前不建议继续触发；
建议保留以下机会；
若预期性价比降至 X 或机会偏好调整为 Y，才会出现可推荐路径。
```
