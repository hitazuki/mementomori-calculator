> 名称校对：优先使用 `doc/items/items.md`、`src/constants/itemScores.json` 与 `data/Master/TextResourceZhCnMB.json` 中已有官方中文。

## 数据口径

- `获得数量` 是单次兑换获得的道具数量，按截图角标读取。
- `限购总数` 是该档位最多可兑换次数；截图中类似 `限购 9/10`、`限购 1/2` 的当前状态统一整理为总限购 `10`、`2`。
- `单次花费` 单位为树影书签。
- 同一道具存在不同兑换档位时保留多行，用 `档位` 区分。
- 截图未显示限购的封印钥匙档位，暂按 `limitTotal = null` 记录，后续实现时可作为不限购或待核验项处理。

## 后续计算建议

后续实现商店兑换性价比时，建议将本表整理为以下字段：

```js
{
  eventKey: 'witch-library-cleanup',
  shopName: '魔女的书库大扫除商店',
  currency: { itemType: 44, itemId: 1, name: '树影书签' },
  reward: { itemType: 16, itemId: 4, name: '命运召唤券', quantity: 5 },
  cost: 375,
  limitTotal: 2,
  tier: 'A'
}
```

计算时可使用：

- `unitCost = cost / reward.quantity`：单个道具的树影书签成本。
- `maxCost = cost * limitTotal`：满购花费；`limitTotal = null` 时不计算满购。
- `maxReward = reward.quantity * limitTotal`：满购获得数量；`limitTotal = null` 时不计算满购。
