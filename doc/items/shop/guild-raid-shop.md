# 公会讨伐战商城

> 来源：用户截图识别，活动页「公会讨伐战」- 商城兑换列表。  
> 兑换货币：活动代币。此活动每次代币名称可能不同；当兑换物品相同或等价时，不区分代币名称，统一视为等价公会讨伐战活动代币。  
> 名称校对：优先使用 `doc/items/items.md`、`src/constants/itemScores.json` 与 `data/Master/TextResourceZhCnMB.json` 中已有官方中文。

## 数据口径

- `获得数量` 是单次兑换获得的道具数量，按截图角标读取。
- `限购总数` 是该档位最多可兑换次数；截图中类似 `限购 10/10` 的当前状态统一整理为总限购 `10`。
- 未显示 `限购` 的档位若来源已确认不限购，使用 `limitTotal = Infinity`；未确认时才使用 `limitTotal = null`。
- `单次花费` 单位为本期公会讨伐战活动代币。
- 同一道具存在不同兑换档位时保留多行，用 `档位` 区分。
- 「伊莱恩召唤券」代表本期特定角色精选召唤券；页面统一显示评分代表名「精选召唤券」，价值按角色召唤模块口径记为 300 钻。
- 「魂瓮残片 伊莱恩」代表本期特定角色专武碎片；游戏文本中没有跨角色通用名，页面按评分代表自行维护为「专属武器碎片」。后续其他角色同类专武碎片可共用同一评分代表。按魔水晶兑换比例 `魔水晶 x3 -> 专武碎片 x10`，单片价值为魔水晶价值的 `3/10`。
- 专武碎片暂不填普通 `ItemType/ItemId`：当前显示图标号 `201` 在 `ItemMB` 中对应 `ItemType=50, ItemId=30` 的活动/聊天贴纸类道具，不是专武碎片本体。实现中继续使用虚拟 `scoreKey` 代表评分与展示。

## CE 计算方案

公会讨伐战商城使用普通商城兑换口径，不把活动代币折算成钻石成本：

```text
RewardValue = 获得数量 × 道具钻石评分
ShopCE = RewardValue / 单次花费
```

展示含义为：每消耗 1 个公会讨伐战活动代币，可换回多少钻石价值。

### 角色专属项

建议新增两个评分代表，避免为每期角色重复建规则：

```js
{
  key: '[16,510]',
  name: '精选召唤券',
  aliases: ['伊莱恩召唤券'],
  score: 300,
  scoreSource: 'itemScores：精选召唤券，代表道具为苏菲亚召唤券',
}

{
  key: 'guild-raid-unique-weapon-fragment',
  name: '专属武器碎片',
  aliases: ['魂瓮残片 伊莱恩'],
  score: 'getScore(13, 1) * 3 / 10',
  scoreSource: '魔水晶商铺兑换比例：魔水晶 x3 -> 专武碎片 x10',
}
```

当前 `itemScores.json` 已包含精选召唤券代表项 `[16,510]`，默认 300 钻；专武碎片仍建议使用虚拟评分键或派生评分规则处理。按当前魔水晶评分 `180` 钻计算，专武碎片单片价值为 `54` 钻。

## 数据结构建议

后续实现商城兑换性价比时，建议将本表整理为以下字段：

```js
{
  shopKey: 'guild-raid',
  shopName: '公会讨伐战商城',
  currency: {
    itemType: null,
    itemId: null,
    name: '公会讨伐战活动代币',
    variableName: true
  },
  products: [
    {
      id: 'guild-raid-001',
      name: '命运召唤券',
      tier: 'A',
      reward: { itemType: 16, itemId: 4, quantity: 1 },
      cost: 75,
      limitTotal: 10,
      source: 'screenshot'
    },
    {
      id: 'guild-raid-003',
      name: '精选召唤券',
      tier: 'A',
      reward: {
        itemType: 16,
        itemId: 510,
        quantity: 1
      },
      cost: 50,
      limitTotal: 10,
      source: 'screenshot'
    },
    {
      id: 'guild-raid-004',
      name: '专属武器碎片',
      tier: 'A',
      reward: {
        itemType: null,
        itemId: null,
        scoreKey: 'guild-raid-unique-weapon-fragment',
        unitScoreFormula: 'getScore(13, 1) * 3 / 10',
        quantity: 10
      },
      cost: 185,
      limitTotal: 10,
      source: 'screenshot'
    }
  ]
}
```

字段规则：

- `currency.variableName = true` 表示不同活动期可显示不同代币名，但 CE 计算时同类代币等价。
- `name` 使用跨期评分代表名，便于排序、筛选和评分复用；截图中的本期角色名仅保留在备注，不在页面商品名中显示。
- `scoreKey` 可指向虚拟评分代表；精选召唤券已可直接使用代表道具 `[16,510]`。
- `limitTotal = null` 表示截图未确认限购；来源确认不限购时使用 `Infinity`。
- `contents` 不存在时直接使用 `reward` 计价；若未来出现礼盒类兑换，再用 `contents` 展开计价。

## 页面展示建议

默认展示顺序使用本表 `序号`，便于对照游戏内列表；页面可额外提供按 `ShopCE`、`RewardValue`、`cost` 排序。

商品卡片/表格建议展示：

- 道具名：显示评分代表名或实际显示代表名。
- 获得数量、单次花费、限购总数。
- `RewardValue`：钻石价值。
- `ShopCE`：每 1 活动代币可换回的钻石价值。
- 评分状态：正常、缺评分、缺价格、限购待确认。
- `档位` 仅用于源数据中区分同一道具的不同兑换价格，用户侧不展示该标签。

详情层建议展示：

- 使用的评分键、单价、数量、总价值。
- 角色专属项显示“同类角色等价代表”，并列出本期截图名。
- 缺评分项显示在 `missingScoreItems` 中，不静默隐藏。
