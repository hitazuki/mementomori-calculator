# 讨伐拉表技能结构化口径

本文记录讨伐倍率拉表当前采用的技能描述、状态计数和行动时序口径，供后续增加角色时参考。

具体字段、枚举值、已实现词条和复用示例见 [skill-term-reference.md](./skill-term-reference.md)；游戏内指南、MB字段和标准词条的对应关系见 [official-battle-guide.md](./official-battle-guide.md)。本文件负责说明“项目采用什么结算口径”，词条目录负责说明“角色数据如何填写”，官方语义文档负责说明“文本和MB应如何解释”。

适用范围是“基础攻击力归一为 1、单体木桩、只计算防御前倍率”的配队规划功能。本文不是完整战斗模拟规范；若新角色依赖死亡、血量、暴击、敌方行动等机制，应先确认是否扩展功能边界。

## 1. 信息来源与优先级

新增角色时按以下顺序核对数据：

1. `ActiveSkillMB.json`、`PassiveSkillMB.json`：技能等级、专武档位、冷却、描述和子技能组成。
2. `EffectGroupMB.json`：EffectGroup 名称、是否隐藏以及可解除性质的辅助说明。
3. `SkillDescriptionLinkTextMB.json` 与游戏内指南：把脱力、迟缓等标准词条展开为官方通用语义。
4. `TextResource*MB.json`：官方角色名、技能名、状态名和 `HelpMainText`，不自行翻译游戏专有名词。
5. 战斗日志：确认实际发动顺序、目标、段数、状态类别、持续时间补偿和历史计数。
6. 项目抽象规则：以上来源无法证明、但拉表必须确定的行为，采用本文注明的固定约定。

描述文本用于理解意图，战斗日志用于确认运行时行为。不能仅凭日志中某个嵌套 `SubSkillResult` 的 EffectGroup 判断施法者；敌方偷取、复制、反击等结果也可能嵌套在当前行动内，应同时检查 `AttackUnitGuid`、`TargetUnitGuid`、`GranterGuid` 和所属行动阶段。

## 2. 伤害倍率统一口径

### 2.1 基础单位

- 全员基础攻击力归一为 `1`，ATK 伤害统一表示为 `%ATK`。
- 无法换算为 ATK 的腕力、魔力等直伤使用 `%STR`、`%MAG` 等符号项单列，不并入 ATK 总计。
- 普通攻击统一为 `100% ATK`，保留物理/魔法标签，但标签暂不改变倍率。
- 计算结果包含配置的暴击倍率，但仍是防御、贯通结算前的理论倍率，不等于实际面板伤害。

### 2.2 段数与目标数

- 明确写明的多段攻击完整保留：`5次攻击力×640%` 结构化为 `{ percent: 640, hits: 5 }`。
- “随机敌人5体”“敌人全体”等多目标攻击，对单体木桩只结算一次，并用 `originalTargetCount` 保留原目标数供详情展示。
- 每次命中都可独立触发追加攻击的技能，只有在触发条件纳入当前模型时才增加段数。
- 击杀追加、暴击追加等只有在触发条件纳入模型时才生成额外伤害段；未模拟的追加效果写入 `ignoredKeys`。

### 2.3 倍率渠道

当前每个主动伤害段的有效倍率公式为：

```text
有效倍率 = 基础倍率
         × (1 + 百分比攻击增幅 + 面板引用符号项)
         × (1 + 角色输出增伤 + Boss承伤增加 - 相关减伤)
         × 暴击倍率
```

- 角色输出增伤与Boss承伤增加属于同一增伤渠道，先相加再作为一个乘区。
- 同一 EffectGroup 可以同时提供多个渠道，例如佛罗伦斯的常驻攻击与增伤；EffectGroup 数仍只算一个。
- 默认所有主动攻击伤害必定暴击，基础暴伤加成为110%，即最终暴击倍率2.10；技能和专武暴伤 Buff加到110%中。
- 命中、闪避、反伤、防御和贯通等仍不进入当前倍率公式。

## 3. EffectGroup 与 Buff 数

### 3.1 日志类别

目前从日志确认的 `SkillCategory` 含义如下：

| SkillCategory | 含义 | 当前引擎处理 |
| --- | --- | --- |
| `2` | 可解除的正面 Buff | `removableBuff`，计入 Buff 数 |
| `3` | 可解除的负面效果 | `removableDebuff`，Boss易伤等效果参与模拟 |
| `4` | 不可解除的正面状态 | `unremovableState`，不计入 Buff 数 |
| `5` | 不可解除的负面状态 | 当前木桩模式不模拟 |
| `200/201` | 内部条件、次数或流程标记 | 不作为普通状态展示，不计入 Buff 数 |

`EffectTurn=9999`、`IsHide` 和 `RemoveEffectType` 不能单独用于判断是否可解除。当前判断应优先依据日志 `SkillCategory`，再用技能描述和 EffectGroup 文本交叉验证。

### 3.2 Buff 数的计数单位

- Buff 数按目标身上的可解除 EffectGroup 数量统计，不按属性条目数量统计。
- 同一 EffectGroup 同时增加攻击和防御，只算一个 Buff。例：茉缇 S2 的攻击+85%、防御+65%。
- 同一技能施加两个独立 EffectGroup，应算两个 Buff。例：梅琳 S1 的攻击 Buff 与暴击伤害 Buff。
- 即使效果属性不参与伤害计算，只要是可解除正面 EffectGroup，仍计入 Buff 数。例：暴击抗性、吸血、可解除护盾。
- 不可解除护盾、守护、不可解除攻击状态不计入 Buff 数。

上述“只统计可解除 Buff”已由鹭丝堤卡日志验证：可解除 Buff 为 3 个且存在多个不可解除状态时，未触发“4个以上 Buff”的被动；可解除 Buff 达到 4 个时触发。

### 3.3 刷新与同组效果

当前运行态以 `(targetId, effectGroupId, sourceId)` 识别一个状态。同一来源重复施加同一组时刷新持续次数，不重复增加 Buff 数；不同 EffectGroup 可同时存在并叠加。

如果新角色涉及以下情况，不能直接沿用，需要先补日志：

- 两个不同角色施加相同 EffectGroupId。
- 同组效果允许叠层、层数独立衰减或只替换数值。
- 可解除 Buff 被偷取、复制、延长或缩短后是否保留原 `GranterGuid`。

## 4. 行动阶段与持续次数

引擎按以下阶段执行一次角色行动：

1. `battleStart`：开战状态。
2. `actionStart`：行动开始被动和条件判断。
3. 选择 S1、S2 或普通攻击，并设置技能冷却。
4. `beforeDamage`：自损事件、伤害前 Buff、Boss状态、目标选择和前置效果。
5. 按 `damageSteps` 逐段、逐命中生成状态快照并结算；暴击后触发效果在当前段之后、下一段之前生效。
6. `afterDamage`：全部伤害段结束后的效果、纯辅助技能和冷却缩减。
7. 当前角色技能冷却各减 1；消耗本次行动应计时的状态。
8. `actionEnd`：在状态持续次数减少后触发行动结束被动。

持续时间按“目标行动次数”处理：

- 目标行动开始前已经存在的状态，覆盖本次行动并在行动结束后消耗 1 次。
- `actionStart` 施加给当前行动者的 1 次行动状态作用于本次伤害，并在本次行动后到期。
- 技能执行过程中施加给当前行动者的 N 次行动 Buff，不消耗施加当次，之后完整覆盖 N 次行动。
- 目标本回合已经行动后才获得 Buff，也从下一次行动开始覆盖完整次数。

日志中的 `IsExtendEffectTurn=true` 和“描述4回合、日志写入 EffectTurn=5”是对已行动目标的回合补偿，不代表技能实际持续5次目标行动。

## 5. 目标选择与排序

当前支持以下选择器：

| 选择器 | 当前含义 |
| --- | --- |
| `self` | 施法者本人 |
| `adjacent` | 线性站位左右相邻角色 |
| `topAttack` | 使用页面手动配置的固定攻击优先级 |
| `highestBuffCount` | 可解除 Buff 数最多的友军，可包含自己 |
| `highestBuffCountOther` | 可解除 Buff 数最多的其他友军 |
| `lowestSpeedOther` | 效果施加前速度最低的其他友军 |

统一约定：

- 攻击优先级不随战斗中的攻击力 Buff 动态变化。
- Buff 数在技能实际选目标的阶段读取；技能本次即将施加的 Buff 不参与自己的目标选择。
- Buff 数并列时按当前站位从前到后选择。
- “相邻1人”在左右两人都存在时，也按站位从前到后选择第一人。
- 每回合开始时按有效速度生成并冻结行动顺序；同速时按站位。行动中的速度变化从下一回合生效。

后两条是为了让规划结果确定而采用的项目约定，尚未由受控日志证明是游戏原生规则。

## 6. 冷却与技能历史

### 6.1 冷却

- 两个主动技能同时可用时优先 S1。
- 主动技能发动后设置最大冷却，在角色本次行动结束时减 1。
- 冷却缩减作用于目标所有当前处于冷却中的主动技能，最低减到 0。
- 冷却缩减发生在技能结算后的角色，可以改变其下一次行动；发生在尚未行动的角色，可以改变其本回合将使用的技能。

### 6.2 技能历史

- “该技能每发动一次……”使用角色运行态中的技能发动次数，不建模为可见 Buff。
- 当前技能使用 `usesBefore` 决定本次效果，技能主体结算完成后再将使用次数加 1。
- 鹭丝堤卡 S2 已由日志确认：第 1～4 次分别为 `4、5、6、6` 段，即：

```text
hits = min(4 + S2历史发动次数, 6)
```

- 内部历史不会被解除，不计入 Buff 数，也不展示为普通状态。

若新技能按“战斗中所有主动技能发动次数”“任意角色发动次数”“命中次数”或“成功触发次数”增长，应单独增加对应计数器，不能复用单技能 `skillUses`。

## 7. 当前木桩假设

- 木桩永久存活、不行动、无防御、无护盾且保持高HP，但可承受拉表角色施加的异常状态。
- 己方可选择1至5人，永久存活并保持满血；自损只产生技能触发事件，不实际降低生命。
- 默认所有主动攻击段必定暴击；闪避、命中、反伤、承伤、死亡、防御、贯通和敌方行动仍忽略。
- 因“满血且无弱化”，梅琳 S1 施加攻击与暴击伤害 Buff，不走治疗或净化分支。
- 因“木桩无护盾”，光士 S2 使用无护盾时的 1.5 倍直伤分支。
- 护盾数值和承伤不计算，但可解除护盾仍作为 Buff 数状态存在。
- Boss状态按全局回合结束计时；光士易伤无持续时间，沙尘等有持续回合的状态正常到期。
- 敌方偷取、复制、延迟等日志行为不进入木桩模拟。

## 8. 推荐的角色定义结构

普通固定伤害段：

```js
{
  key: 's1',
  cooldown: 4,
  damageType: 'phys',
  damageSteps: [
    { stat: 'ATK', percent: 640, hits: 5, damageType: 'phys' },
  ],
  hooks: [],
}
```

运行时状态：

```js
statusEffect({
  id: 'character-skill-atk',          // 项目内稳定标识
  effectGroupId: 123456789,           // MB/日志中的 EffectGroupId
  nameKey: 'raidBuffCharacterAttack',
  target: 'highestBuffCount',
  duration: 4,
  statusClass: RAID_STATUS_CLASSES.REMOVABLE_BUFF,
  modifiers: [
    { id: 'character-skill-atk', channel: 'attackRate', rate: 0.3 },
  ],
}) // 放入 hook('beforeDamage', [该效果])
```

按技能历史变化的段数：

```js
{
  stat: 'ATK',
  percent: 740,
  hits: {
    type: 'skillUsesLinear',
    skillKey: 's2',
    base: 4,
    increment: 1,
    max: 6,
  },
}
```

没有倍率作用但影响 Buff 数的状态，`modifiers` 使用空数组，不能省略整个 EffectGroup。

跨面板加攻不换算成普通攻击百分比，而保留符号项：

```js
symbolicModifiers: [
  { kind: 'sourceAttackOverTargetAttack', coefficient: 0.5, sourceId: 93 },
  // 或 { kind: 'targetBaseDefenseOverTargetAttack', coefficient: 2 }
]
```

例如阿尔托莉亚为目标提供 `0.5×ATK_93/ATK_目标`，利伯瑞娅和静提供 `2×DEF0_目标/ATK_目标`。开战跨面板效果统一读取未受其他开战 Buff 影响的原始面板。

## 9. 已确认规则与待确认规则

### 已由官方指南、MB 与日志确认

- 可解除正面 Buff 使用 `SkillCategory=2`，不可解除正面状态使用 `SkillCategory=4`。
- 鹭丝堤卡的 Buff 数条件只统计可解除 EffectGroup。
- S1/S2 的目标 Buff 在主体伤害前施加；选中自己时，本次 S2 立即获得攻击加成。
- 描述4次行动的自身技能 Buff 不在施加当次被消耗。
- 鹭丝堤卡 S1 固定5段，S2 历史段数为 `4→5→6→6`。
- 鹭丝堤卡行动开始增攻属于不可解除状态，不反过来增加自己的 Buff 数。
- 鹭丝堤卡护盾每8次自身行动触发，首次为第1次行动、下次为第9次行动。
- 梅琳 S1 的攻击与暴击伤害是两个 EffectGroup；茉缇 S2 的攻击与防御属于一个 EffectGroup。
- 光士每个暴击伤害段结算后叠加一层15%易伤，当前段只使用之前的层数；Boss易伤与佛罗伦斯输出增伤加算。
- 暴击开启时佛罗伦斯 S1 从初始6段追加到10段。
- 行动顺序只在回合开始时按速度结算，本回合内不动态重排。
- “效果持续N回合”通常按附带者的行动结束次数计时；行动过程中获得的状态从下一次行动结束开始消耗。
- “第N回合、回合开始/结束、本回合”等属于全局TURN，不能与角色行动次数混用。
- 能力值排序目标同值时按站位顺序；连续攻击每段重新排序。战斗开始及第1回合开始的排序使用状态生效前能力值。
- 无法被解除的状态不计入角色身上的增益/弱化总数，但仍可满足特定状态条件。

### 当前项目约定，尚未完全验证游戏原生行为

- Buff 数并列时按站位顺序。
- 中间站位的“相邻1人”按站位顺序选择。
- 同一来源、同一 EffectGroup 若未声明重附加策略，当前引擎按整体替换处理；官方实际规则应区分共存、替换、保留最强最长和叠层。
- 攻击力最高目标使用页面固定优先级，不动态比较 Buff 后攻击力。
- 概率效果采用可切换的成功/失败确定场景，不计算概率期望。
- Boss不行动时，有持续时间的Boss状态暂按全局回合结束计时；这是木桩模式抽象，不等同官方一般状态计时规则。

### 增加复杂角色前应优先补充日志的情况

- 等 Buff 数目标的游戏原生并列规则。
- 随机目标、多次随机目标是否允许重复命中同一目标。
- 同组叠层、层数消耗、不同来源同组覆盖与刷新。
- Buff 延长、缩短、偷取、复制、驱散的选择顺序和来源归属。
- 技能文本未明确属于全局TURN还是附带者行动次数时的具体判定，尤其是Boss不行动场景。
- 反击、追击、协同攻击是否算主动技能历史或触发其他被动。
- HP、死亡、复活、召唤物、替身或队伍人数动态变化。
- 暴击成功、命中成功、异常状态成功率等概率条件。
- 攻击前后同时存在多组直伤、ATK 伤害和状态施加时的精确顺序。

## 10. 新角色接入检查清单

1. 确认角色 ID、Lv240 最高技能、Ex3 子技能列表和最大冷却。
2. 从五种 `TextResource*MB` 取得官方角色名与技能名。
3. 将每个伤害段拆成有序 `damageSteps`，记录 `stat / percent / hits / damageType / originalTargetCount`。
4. 标出每个效果的触发阶段：开战、行动开始、伤害前、伤害后或行动结束。
5. 为每个运行时状态记录真实 EffectGroupId、可解除类别、持续次数和目标选择器。
6. 即使属性被忽略，也确认该 EffectGroup 是否会影响 Buff 数或其他技能条件。
7. 明确技能历史在本次结算前还是结算后递增。
8. 列出当前木桩口径忽略的分支，不能默默删除。
9. 至少补充以下测试：技能轮转、逐段暴击、Boss状态前后快照、目标选择、速度顺序、Buff 时序、冷却变化、状态到期和默认阵容回归。
10. 运行 `npm test` 与 `npm run build`，并在页面中切换替补、站位和优先级进行一次交互验证。

## 11. 单体木桩中的全体与弱化数量

- “全体友军”按当前上阵名单依站位顺序施加；再由属性筛选缩小目标时，筛选发生在全体选择之后。
- “目标附带的弱化效果个数”在单体木桩中按不同的 Boss EffectGroup 状态组计算，同一状态组的层数仍只算一个弱化效果。
- 这两个映射仅用于把多目标技能投影到单体木桩，不改变实际的伤害段数或随机目标语义。

## 12. 零倍率弱化组与动态状态值

- 防御、物防、回避或受治疗量等暂不进入木桩倍率的弱化，仍须用独立 Boss EffectGroup 建模，并将 `damageRatePerStack` 设为 `0`。
- 这种零倍率弱化会按正常 Boss 回合时钟刷新和到期，可供“每有一个弱化效果”类技能读取；同一 EffectGroup 的层数仍只算一个弱化。
- 需要随来源角色计数器或木桩弱化数量变化的状态 Modifier，必须通过已注册的 value resolver 在读取快照时求值，不得在角色模块中写解析函数。
